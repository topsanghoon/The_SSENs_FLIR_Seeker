# The_SSENs_FLIR_Seeker 개발 가이드라인

이 문서는 **The_SSENs_FLIR_Seeker 프로젝트 팀원들**이 일관성 있고 효율적으로 개발하기 위해 따라야 할 규칙을 정리한 가이드라인입니다.  
외부 공개보다는 내부 협업 기준에 초점을 맞추었습니다.

---

## 📂 디렉토리 규칙
- `src/firmware/` : STM32, Zynq 등 임베디드 보드용 코드
- `src/host/` : PC/Linux 실행용 코드 (Python, C++ 등)
- `src/sim/` : 시뮬레이션 및 알고리즘 검증 코드
- `hardware/` : 회로도, 보드 핀맵, 센서 연결 정보
- `docs/` : 기술 문서, 회의록, 설계 문서
- `data/` : 테스트용 열화상 데이터, 로그 (실제 민감 데이터 업로드 금지)

⚠️ 디렉토리 구조는 반드시 유지하고, 새로운 파일은 해당 카테고리에 맞게 추가합니다.

---

## 🧑‍💻 코드 스타일

### C / C++ (펌웨어 및 알고리즘)
- 들여쓰기: **4 spaces**
- 파일명: `snake_case` (예: `flir_parser.c`, `tracking_algorithm.cpp`)
- 함수명: `lowerCamelCase` (예: `processFrame()`)
- 클래스명(C++): `PascalCase` (예: `TargetTracker`)
- 상수/매크로: 대문자 스네이크 (예: `MAX_FRAME_SIZE`)
- 헤더 파일은 `#pragma once` 권장

### 문서 (Markdown)
- 제목 레벨은 3단계까지만 사용
- 코드 블록은 언어 지정 필수 (예: \`\`\`cpp, \`\`\`python)

## ✅ 커밋 규칙
- 커밋 메시지는 다음 접두어를 사용:
  - `Add:` 새 기능 추가
  - `Fix:` 버그 수정
  - `Update:` 기존 코드/문서 개선
  - `Refactor:` 구조 변경 (기능 변화 없음)
  - `Docs:` 문서 관련 변경
  - `Test:` 테스트 추가/변경
  - `Chore:` 환경 설정, 빌드 스크립트 등 기타

예시:

```
Add: STM32 UART DMA 기반 FLIR 데이터 수집 모듈
Fix: Python 시뮬레이션 NaN 처리 버그 수정
```

## 📊 테스트 & 검증
